 \hypertarget{md_doxygen_doc_internal_commands_internal_cmdilinebr}{}\doxysection{\textbackslash{}ilinebr}\label{md_doxygen_doc_internal_commands_internal_cmdilinebr}
\index{\textbackslash{}@{\textbackslash{}}} Internal doxygen command to simulate an end of line, but without advancing the line counter. In this way it is possible to have multiple commands on one line that read till the end of line. This command is internally used by doxygen and for replacement of the {\ttfamily $^\wedge$$^\wedge$} in {\ttfamily ALIASES} settings. This command sees to it that e.\+g. warning messages stay correct when a command is replaced internally by multiple commands.

Temporarily also as {\ttfamily @\+\_\+ilinebr} (doxygen version 1.\+8.\+14) and {\ttfamily \textbackslash{}\+\_\+ilinebr} (doxygen version 1.\+8.\+15 till 1.\+8.\+18)

\begin{DoxySince}{Since}
doxygen version 1.\+8.\+19
\end{DoxySince}

 \hypertarget{md_doxygen_doc_internal_commands_internal_cmdiliteral}{}\doxysection{\textbackslash{}iliteral\mbox{[}\textquotesingle{}\{\textquotesingle{}$<$option$>$\textquotesingle{}\}\textquotesingle{}\mbox{]}}\label{md_doxygen_doc_internal_commands_internal_cmdiliteral}
\index{\textbackslash{}iliteral@{\textbackslash{}iliteral}}

This command is to replace the Java documentation commands {\ttfamily \{@literal .... \}} and {\ttfamily \{@code ...\}}. The text in the blocks will not be interpreted by doxygen in any way. The text in the {\ttfamily \{@literal} will put as is text in the output. The text in the {\ttfamily \{@code} will be replaced by a code block with class {\ttfamily Java\+Doc\+Code} i.\+e. {\ttfamily \texorpdfstring{$<$}{<}code class=\char`\"{}\+Java\+Doc\+Code\char`\"{}\texorpdfstring{$>$}{>}...\texorpdfstring{$<$}{<}/code\texorpdfstring{$>$}{>}}.
